A Comprehensive Bash Script for Web Reconnaissance

In the vast landscape of the internet, security remains a critical concern for website owners and users alike. Understanding the foundational aspects of a website's security posture is paramount in safeguarding sensitive data and ensuring a trustworthy online presence.

To shed light on this crucial facet of web security, I've crafted a comprehensive script - a tool designed for reconnaissance purposes. This script aims to provide a structured approach to probing and gathering essential information about a website's infrastructure, all orchestrated through a series of systematic steps.

Exploring the Script's Capabilities

This Simple Web Recon Script combines multiple techniques into a seamless workflow, providing a comprehensive analysis of website security.

DNS Enumeration:

Unveiling the domain's DNS records - the vital infrastructure map of a website. The script initiates the extraction of Domain Name System (DNS) records associated with the target domain or IP address. This phase aims to uncover essential information about the domain's infrastructure, such as IP addresses, mail servers (MX records), canonical names (CNAME records), and text-based records (TXT records).

WHOIS Look Up:

Gathering essential ownership and registration details about the domain by querying WHOIS databases. This step provides insights into administrative contacts, registration dates, and DNS server information, aiding in understanding the domain's ownership history.

Port Scanning:

Probing for open ports and services to assess potential vulnerabilities. This step aids in identifying active services, potentially vulnerable ports, and service version information, critical for vulnerability assessment and system hardening.

Web Server Information:

Uncovering details about the web server, including HTTP/HTTPS responses and SSL certificates. It analyzes server headers to discern details about the web server, such as response codes, server type/version, and potential redirections.

SSL Certificate Information (HTTPS):

Utilizes openssl to inspect and present detailed information about the SSL/TLS certificate associated with the HTTPS connection. This phase reveals certificate details like issuer information, validity period, and cryptographic specifications.

Robots.txt & Directory Listing:

Examining permissions and directory structures for potential vulnerabilities, ensuring secure access control.

An Educational Resource

This article accompanies the script to delve into its functionality, showcasing how each step contributes to a comprehensive security assessment. It provides insight into the commands and tools leveraged within the script, aiming to educate and empower individuals interested in understanding web security reconnaissance. Let's embark on a journey through each stage of the script, unraveling the significance of DNS records, port scanning, web server responses, and SSL certificates in safeguarding the digital realm.

Functions Explanation

1 // Banner Display Function

This part of the script serves the purpose of creating an informative and visually appealing banner that provides an overview of the script's capabilities. It utilizes color-coded text to enhance readability in the terminal. Let's explore its components:

¬∑ Color Definitions: ANSI escape sequences define various colors used in the terminal output.

¬∑ display_banner() Function: This function generates an illustrative banner with descriptive text about the script's functionalities. It utilizes specific colors for readability. Invoking display_banner() prints the informative banner to the terminal when the script is executed.


2 // User Input Validation

This section of the script is responsible for collecting user input regarding the IP address or domain name targeted for reconnaissance. Let's break down its functionality:

¬∑ read -p "Enter IP or domain: " target: Prompts the user to input an IP address or domain name using the read command. The provided input is stored in the target variable.

¬∑ Input Validation: The script checks whether the user has provided valid input (i.e., the target variable is not empty).

¬∑ Error Message and Exit: If the input is invalid (empty), an error message in red color is displayed, indicating that the input was invalid. ¬∑ The script then exits with a status code of 1, halting further execution.

3 // DNS Enumeration

This segment encompasses DNS Enumeration, a fundamental step in discovering critical information about a domain's infrastructure. Let's examine the code section responsible for this enumeration:

¬∑ DNS Enumeration Prompt: The script initiates DNS Enumeration and displays a message indicating this action in yellow color using echo -e.

¬∑ host "$target": Utilizes the host command to perform an initial DNS query for the provided IP or domain stored in the target variable.

¬∑ Array of DNS Record Types: Defines an array record_types containing specific DNS record types: A, AAAA, CNAME, MX, and TXT. Loop through Record Types: Iterates through each DNS record type in the record_types array.

¬∑ Performing nslookup: Executes nslookup for each record type and the provided target (domain or IP). It filters the output using grep -v to exclude lines containing 'NXDOMAIN' (Non-Existent Domain) or 'SERVFAIL' (Server Failure) errors.

4 // WHOIS Lookup

This portion of the script initiates a WHOIS lookup to gather essential registration and ownership details related to the specified domain. Let's dissect this code section:

¬∑ WHOIS Lookup Prompt: The script announces the commencement of a WHOIS Lookup, displaying a message in yellow color using echo -e.

¬∑ whois "$target": Executes the whois command, querying WHOIS databases for detailed information related to the domain or IP stored in the target variable.
¬∑ Displaying WHOIS Information: Prints the retrieved WHOIS information to the terminal.

5 // Port Scanning and Exporting Results

This section of the script conducts a port scanning operation using nmap and exports the results to a text file. Let's break down this code section:

¬∑ Port Scanning Prompt: The script initiates Port Scanning and displays a message in yellow color using echo -e.

¬∑ nmap Command: Executes the nmap command with specific options:
  -sC: Performs default script scanning.
  -sV: Attempts to determine service versions.
  -oN "nmap-scan-$target.txt": Saves the scan results in a text file named "nmap-scan-$target.txt" (where $target is the provided domain or IP address).

¬∑ "$target": Specifies the target IP address or domain name to be scanned.

6 // Web Server Information Retrieval (HTTP + HTTPS)

This section of the script focuses on obtaining essential details about the web server's response for both HTTP and HTTPS connections. Let's dissect this code segment:

¬∑ HTTP Information Fetching: Initiates the retrieval of web server information for HTTP connections. It displays a message in yellow color using echo -e.

¬∑ curl -I "http://$target": Executes curl to send a HEAD request (-I flag) to the specified HTTP target (http://$target) to retrieve the server's response headers.
    2>/dev/null: Redirects error messages to /dev/null, discarding them for a cleaner output.
    | head -n 1: Pipes the output to head command to display only the first line of the response headers.

¬∑ HTTPS Information Fetching: Initiates the retrieval of web server information for HTTPS connections. It also displays a message in yellow color using echo -e. The rest of the command is the same as in the HTTP part.

7 // SSL Certificate Information Retrieval (HTTPS)

This section of the script is dedicated to fetching and displaying detailed SSL certificate information associated with the HTTPS connection. Let's break down this code segment:

¬∑ SSL Certificate Fetching Prompt: The script prompts the retrieval of SSL Certificate Information for HTTPS connections, displaying a message in yellow color using echo -e.

¬∑ openssl s_client Command: Initiates an SSL/TLS connection to the specified target ("$target:443"), where 443 is the default HTTPS port.
    -showcerts: Instructs OpenSSL to display detailed certificate information during the SSL handshake.
    </dev/null 2>/dev/null: Provides an empty input stream and redirects errors to /dev/null to prevent unnecessary output.

| ¬∑ openssl x509 -noout -text: Pipes the output from the SSL connection to openssl x509, which extracts and displays detailed textual information about the SSL certificate.

8 // Robots.txt Retrieval (HTTP + HTTPS)

This script section focuses on retrieving the robots.txt file for both HTTP and HTTPS connections. Let's break down this code segment:

¬∑ Robots.txt Retrieval (HTTP): Begins the retrieval of the robots.txt file for HTTP connections and displays a message in yellow color using echo -e.

    curl -sL "http://$target/robots.txt": Utilizes curl to fetch the robots.txt file for HTTP connections (http://$target/robots.txt). The -s flag ensures silent mode, and -L follows redirections.
     2>/dev/null: Redirects error messages to /dev/null, discarding them for a cleaner output.

¬∑ Robots.txt Retrieval (HTTPS): Commences the retrieval of the robots.txt file for HTTPS connections, displaying a similar yellow-colored message.
    curl -sL "https://$target/robots.txt": Uses curl to obtain the robots.txt file for HTTPS connections (https://$target/robots.txt).

9 // Directory Listing Check for HTTP and HTTPS

The script section below focuses on conducting a directory listing check for both HTTP and HTTPS connections. It encompasses a comprehensive approach to assess the status of directory listing for a given target.

This script segment performs a dual-check for directory listing status over both HTTP and HTTPS connections.

It first sends HTTP and HTTPS requests to the target server's root directory and examines the HTTP response code. If the response code is 200, it implies that directory listing is enabled for the respective protocol. If not, the script proceeds with a directory brute-forcing attempt using dirb along with the -f flag for fine-tuning, aiming to discover potential hidden directories despite possible redirections or unstable responses encountered during the process.
By integrating these checks into the reconnaissance process, this script aims to provide insights into the configuration and security aspects related to directory listing for both HTTP and HTTPS connections, facilitating a comprehensive assessment of potential information exposure and access control on the target server.

However, it's worth noting that, due to time constraints or other factors, you may choose to deactivate the HTTP scan within this script. To accomplish this, simply comment out or remove the HTTP directory listing check section while retaining the HTTPS assessment. This modification streamlines the reconnaissance to focus solely on HTTPS, which might be a pragmatic decision when time is limited or when the website primarily operates over secure connections.
